利用 Python 建立虛擬環境並安裝 Jupyter

前言

  1. 練習使用虛擬環境：在開發程式時套件的版本，可能因為別的專案用到不同的套件版本而打架，所以利用虛擬環境來避免
  2. 使用jupyter notebook作筆記：為了能夠把之前的程式以及未來的程式當中，做更好的筆記，以免日後用到時，又得回去翻所有的.py檔來複習
  3. 在虛擬環境下安裝並使用jupyter notebook：綜合以上兩點決定以虛擬環境調用jupyter notebook，並且將之前練習檔改成.ipynb檔更好筆記

備註

今天發現在終端機 cd foldername 時，可能會無效，因為cd指令只能導引到同一個磁碟機裡的資料夾目錄下，這時改用 cd/d foldername 就行了，以後都直接用 cd/d 就不怕不會引導到想要的目錄下了 參考清單 這裡1 這裡2 這裡3 這裡4

那麼開始吧~(以python3為例)

Step1: 安裝virtualenv

參考 這裡1 中提到

xxxxxxxxxx
pip3 install virtualenv

Step 2: 建立虛擬環境

首先在終端機裡移動到要建立虛擬環境的大目錄，這裡我是先創一個 D:\pyvirEX（可以參考 這裡2，但還是以 這裡1 為主）

xxxxxxxxxx
cd/d E:\pyvirEX
virtualenv venvpy3

就會建立一個以 python3 直譯器為基底的 venvpy3 資料夾

重要

  • 因為用 pip3 安裝，表示是取用 python3 的 pip 去安裝 virtualenv 套件，打上 virtualenv venvpy3 預設以當前的 python3 版本來建立虛擬環境資料夾
  • 也就是利用 pip 安裝 ( 因為我的系統配置，pip 會引用 python2 的 pip)，所以 pip install virtualenv，則是安裝 python2 版本的套件，使用後建立的虛擬環境資料夾，當然就是使用 python2 直譯器
  • 再來在不同 py2 / py3 的虛擬環境資料夾下，都分別安裝 jupyter，開啟後，就會發現兩者的 notebook，就有不同 kernel 的分別了
  • 假如系統沒有配置好而無法靈活調用 pip> python2 / pip3> python3 的話也不要緊，在 python3 資料夾內，複製一個 python.exe 並改名為 python3.exe 後，就可：
xxxxxxxxxx
python -m pip ...  →代表用 py2 的 pip
pytohn3 -m pip(3) ... →代表用 py3 的 pip

備註：如何在同個電腦終端機配置 python2 及 python3

  1. python2 及 python3 加入 path 時，python2 及 python2\Scripts 必須要在 python3 及 python3\Scripts 的上面，因為上下階層表示取用順序
  2. python3 資料夾內，複製一個 python.exe 並改名為 python3.exe
  3. 再將 python2 的 pip 更新到最新，以及將 python3 的 pip 及 pip3 也更新到最新

python → py2 pip → py2 pip

python3 → py3 pip3 → py3 pip

Step3：啟用虛擬環境模式

開啟終端機並導至 venvpy3 資料夾下的 Scripts，並打上 activate

xxxxxxxxxx
C:\Users\123 > cd/d D:\pyvirEX\venvpy3\Scripts
​
D:\pyvirEX\venvpy3\Scripts > activate
​
(venvpy3) D:\pyvirEX\venvpy3\Scripts >

出現 () 就表示進入了虛擬環境，打上 deactivate 便可離開虛擬環境

Step4：虛擬環境下安裝 jupyter

可參考 這裡3 

xxxxxxxxxx
pip install --upgrade pip
pip install jupyter

Step5：建立資料夾存放ipynb檔案並開啟 jupyter

也就是要在哪個資料夾下開啟 jupyter notebook
參考 這裡4 

xxxxxxxxxx
cd/d D:\pyvirEX\venvpy3
mkdir jupyterpy3
cd/d D:\pyvirEX\venvpy3\jupyterpy3
jupyter notebook

下次呼叫時

xxxxxxxxxx
cd/d D:\pyvirEX\venvpy3\Scripts
activate
cd/d D:\pyvirEX\venvpy3\jupyterpy3
jupyter notebook

若要關閉則在 jupyter notebook 按下 Quit 再關掉瀏覽器，可以回到終端機；到終端機後打上 deactivate 就能離開虛擬環境

雜七雜八

為何在存放 .ipynb檔案中會有 .ipynb_checkpoints資料檔？

每當你創建一個新的 notebook 時，都會多創建一個 checkpoint ；它將位於你保存位置的隱藏子目錄中稱作 .ipynb_checkpoints，也是一個 .ipynb 文件。默認情況下，Jupyter 將每隔 120 秒自動保存你的 notebook，或者可主動按下功能鍵中最左邊的儲存按鈕，notebook 和 checkpoint 文件都將被更新。因此，該文件使你能夠在發生意外事件時恢復未保存的工作。你可以通過 “File > Revert to Checkpoint“ 從菜單恢復到上一個 checkpoint 時的模樣
